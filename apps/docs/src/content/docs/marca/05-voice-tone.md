## Princípios

1. **Direto** — verbo principal primeiro. "Salvar alterações" > "Clique aqui para salvar suas alterações".
2. **Respeitoso** — não condescendente. Erro é informação útil, não punição.
3. **Confiante mas humano** — ações têm verb-first ("Confirmar pagamento", "Excluir tenant"); empático em mensagens de erro.
4. **PT-BR primeira classe** — não tradução literal. "Cancelar" (não "Cancele"), "Senha incorreta" (não "Sua senha está incorreta").

## Microcopy padrão

| Situação               | Texto recomendado                                  |
| ---------------------- | -------------------------------------------------- |
| Estado vazio (lista)   | "Nenhum {item} ainda. [Criar primeiro {item}]"     |
| Erro de validação      | "{Campo} é obrigatório" / "{Campo} inválido"       |
| Confirmação destrutiva | "Excluir {item}? Esta ação não pode ser desfeita." |
| Sucesso                | "{Item} salvo" / "{Item} excluído"                 |
| Loading                | "Carregando…" (apenas se >300ms)                   |
| CTA primário           | Verb-first: "Salvar", "Confirmar", "Criar tenant"  |
| CTA secundário         | "Cancelar" (não "Voltar")                          |

## Don'ts

- ❌ "Por favor, clique no botão para continuar" → ✓ "Continuar"
- ❌ "Ocorreu um erro inesperado" → ✓ "Não foi possível conectar ao servidor"
- ❌ "Você tem certeza que deseja…" → ✓ "Confirmar exclusão?"
- ❌ "Sucesso!" → ✓ "Tenant criado"
